Reality Builder Auto Updater.

Эта программа позволит всем пользователям получать самые последние обновления нашего ПО без необходимости ручного обновления через github. Данная программа загружает последнюю стабильную версию нашего ПО. Пользователи по прежнему могут использовать github чтобы получить самую последнюю версию ПО, но это рекомендуется делать только опытным пользователям. Используя авто обновление вы можете быть уверены в том что вам всегда будет доступна самая последняя поддерживаемая версия нашего ПО.


Скачать Reality Builder Auto updater можно здесь http://abighole.hngamers.com/10kli 

После скачивания и запуска нашей программы она будет установлена в каталог c:\RealityBuilder

База данных 0.40+

Начиная с версии 0.40+ (обновление Lingor) мы частично очистим таблицы техники. Этой очисткой мы пытаемся разрешить множество проблем возникающих со старыми версиями базы данных. Предварительно перед началом обновления любая техника добавленная пользователем должна быть перемещена из за изменения порядковых номеров техникию. Предлагаем вам сменить порядковые номера вашей техники на номера выше 1000 чтобы оставить место для карт которые будут добавлены в будущем.

Мы понимаем что это будет сложно сделать тем администраторам, которые добавили большую часть транпорта своими руками, однако мы полагаем, что так будет лучше для всех пользователей данного ПО в долгосрочной перспективе и позволит быстрее портировать карты в нашу Reality систему.

______________________________________________________________________________________________________________________________
Требования.

1. Операционная система Windows (тестировалась на 7 и Server 2008);
2. Выделенный сервер ArmA 2 Combined Ops (Пользователи Steam должны объединить каталоги ArmA2 и ArmA2 OA) с установленным бета патчем последней версии (http://www.arma2.com/beta-patch.php);
3. Microsoft Visual C++ 2010 SP1 x86 Redistributable (http://www.microsoft.com/en-us/download/details.aspx?id=8328);
4. Microsoft .NET Framework 4 или выше;
5. MySQL Server 5.x с активированным TCP/IP Networking. ПРИМЕЧАНИЕ: Вы должни использовать официальный инсталятор MySQL, а не XAMPP (http://dev.mysql.com/downloads/mysql/);
6. Дясятичным разделителем ОБЯЗАТЕЛЬНО должна быть точка. (https://github.com/thevisad/DayZ-Private-master/wiki/Decimal-Separator);
7. Strawberry Perl 32 битная версия (не 64 битная)>= 5.16 (http://strawberryperl.com/) (64 битная версия не может правильно установить все необходимые зависимости);
8. Клиент DayZ 1.7.7.1 (http://cdn.armafiles.info/latest/1.7.7.1/%40Client-1.7.7.1-Full.rar);

Установка.

ПРИМЕЧАНИЕ: Важно выполнять все шаги из нашей Wiki правильно и последовательно.
ПРИМЕЧАНИЕ: Настоятельно рекомендуется отключить UAC перед началом установки, особенно перед установкой Perl. В данном случае запуск установки от администратора работает некорректно из за специфической установки Perl.

______________________________________________________________________________________________________________________________
Обновление.

1. Запустите update_scripts.pl .\util\dayz_config\BattlEye (для обновления скриптов Battleye);
2. Запустите update_bans.pl --dwarden --banz --cbl --save .\util\dayz_config\BattlEye\bans.txt (для обновления списка банов);
3. Соберите свой сервер через наше ПО.

ПРИМЕЧАНИЕ: Если вы делаете это в обратной последовательности, то скрипты\баны для Battleye не обновятся и вам придется делать это вручную. Данная последовательность позволит предварительно обновить скрипты\баны нашего ПО, перед тем как вставить их в вашу сборке сервера.

Дополнительные возможности.

Во время сборки вашего сервера, вы можете активировать дополнительные пакеты. Список поддерживаемых Reality пакетов:
Название:		Параметр строки: 			Описание:
carepkgs		--with-carepkgs 			Разбрасывает ящики с разнообразным типом лута по всей карте (Схоже с хеликрашами).
killmsgs		--with-killmsgs				Показывает в игре сообщения о смерти игроков. Должны быть использованы измененные BattlEye фильтры (Необходимо предварительно их скачать).
messaging		--with-messaging			Через определенный интервал времени отправляет заданные сообщения от имени сервера.
buildings		--with-buildings			Позволяет вставлять строения через базу данных.
wrecks			--with-wrecks				При старте сервера разбрасывает обломки техники с лутом по всей карте.
invcust			--with-invcust				Позволяет изменять стартовый набор снаряжения для индивидуального игрока или группы.
mbg_celle2		--with-mbg_celle2			Модификация\фикс Server-pbo для карты celle.
ssZeds			--with-ssZeds				Отключает симуляцию зомби на стороне сервера (может улучшить FPS сервера).
ssZedsMessaging	--with-ssZedsMessaging		Отключает симуляцию зомби на стороне сервера (может улучшить FPS сервера) плюс пакет Messaging соединенные вместе. Если вам нужны оба этих пакета следует использовать именно этот, чтобы избежать конфликтов двух данных пакетов.
